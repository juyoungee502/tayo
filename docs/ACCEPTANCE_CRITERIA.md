# 타요 구현 완료 기준

Codex는 아래 항목을 만족해야 작업을 완료로 간주한다.

## A. 실행 가능성
- `npm install` 후 개발 서버가 실행된다.
- `npm run lint`가 통과한다.
- `npm run build`가 통과한다.
- README에 실행 방법이 정리되어 있다.

## B. 인증
- 로그인 페이지가 존재한다.
- `@catholic.ac.kr` 이메일만 로그인 시도를 허용한다.
- 최초 로그인 시 프로필이 생성된다.
- 로그인 전 보호 라우트 접근 시 로그인 페이지로 유도된다.

## C. 택시팟 생성/목록/상세
- 택시팟 생성 폼이 동작한다.
- 생성자 자동 참여가 처리된다.
- 택시팟 목록이 보인다.
- 지역/시간 기반 필터가 최소 수준으로 동작한다.
- 상세 페이지에서 참여/취소가 가능하다.
- 정원 초과와 중복 참여가 막힌다.
- 사용자는 동시에 하나의 활성 팟에만 속할 수 있다.

## D. 카카오맵
- 지도 키는 `NEXT_PUBLIC_KAKAO_MAP_APP_KEY`에서만 읽는다.
- 키가 있으면 Kakao 지도 검색/선택 UI가 동작한다.
- 키가 없거나 SDK 로드 실패 시 수동 텍스트 입력 UI가 자동 대체된다.
- 지도 관련 로직은 재사용 가능한 컴포넌트/유틸로 분리된다.

## E. 피드백/신고
- 탑승 후 피드백 페이지가 있다.
- 후기 입력이 가능하다.
- 특정 사용자 신고가 가능하다.
- 신고 데이터는 관리자 화면에서 조회 가능하다.
- 중복 제출 방지 로직이 있다.

## F. 마이페이지
- 내 프로필 정보가 보인다.
- 이용 히스토리가 보인다.
- 기본 정보 수정 기능이 있다.
- 탈퇴 기능 또는 최소한의 탈퇴 흐름이 제공된다.

## G. 관리자
- `/admin` 보호 라우트가 있다.
- 관리자만 접근 가능하다.
- 사용자 / 택시팟 / 신고 목록을 볼 수 있다.

## H. 문서화
- `.env.example`가 있다.
- Supabase 스키마/마이그레이션이 있다.
- README에
  - 실행 방법
  - 환경변수 설명
  - Kakao 지도 설정 방법
  - 관리자 계정 부여 방식
  - 수동 테스트 체크리스트
  가 포함되어 있다.

## I. 품질
- TypeScript 타입 오류가 없다.
- 큰 컴포넌트는 적절히 분리된다.
- 에러 상태 / 로딩 상태 / 빈 상태가 최소 1회 이상 고려된다.
- 비밀키 하드코딩이 없다.
