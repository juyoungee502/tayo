당신은 현재 저장소에서 작업하는 시니어 풀스택 엔지니어입니다. 이 작업은 **가톨릭대학교 성심교정 학생 대상 택시 합승 웹 MVP '타요'** 구현입니다.

작업을 시작하기 전에 반드시 아래 순서대로 진행하세요.

1. `AGENTS.md`를 읽고 준수하세요.
2. `docs/PRODUCT_SPEC.md`를 읽으세요.
3. `docs/ACCEPTANCE_CRITERIA.md`를 읽으세요.
4. 현재 저장소 구조를 점검하세요.
5. 작업 전에 짧은 구현 계획을 먼저 제시하세요.
6. 그 다음 실제 구현을 시작하세요.

## 최종 목표
다음 기능이 실제로 동작하는 웹 MVP를 구현하세요.

- 학교 이메일(`@catholic.ac.kr`) 기반 로그인
- 프로필 생성
- 택시팟 생성
- 택시팟 목록/상세
- 택시팟 참여/취소
- 한 사용자는 동시에 하나의 활성 택시팟만 참여 가능
- 모임 후 후기/신고 작성
- 마이페이지
- 관리자 보호 페이지
- Kakao 지도 기반 출발지 검색/선택
- 지도 실패 시 수동 텍스트 입력 fallback

## 구현 원칙
- Next.js App Router + TypeScript + Tailwind를 사용하세요.
- Supabase를 auth + database로 사용하세요.
- 코드 품질과 유지보수성을 우선하세요.
- 비밀값은 하드코딩하지 마세요.
- 카카오 지도 키는 반드시 `NEXT_PUBLIC_KAKAO_MAP_APP_KEY`에서만 읽으세요.
- Kakao SDK는 클라이언트에서만 로드하세요.
- SDK 로더는 재사용 가능한 유틸/훅/컴포넌트로 분리하세요.
- Kakao SDK 로드 URL에는 `autoload=false&libraries=services`를 포함하세요.
- 키가 비어 있거나, 도메인이 등록되지 않았거나, SDK가 실패해도 앱 전체는 정상 동작해야 합니다.
- 그 경우 수동 출발지/도착지 입력 UI로 자동 대체하세요.

## 인증/권한
- `@catholic.ac.kr` 이메일만 로그인 가능하게 구현하세요.
- 최초 로그인 시 `profiles` row를 생성하세요.
- 관리자 계정은 공개 회원가입에서 만들 수 없게 하세요.
- `/admin`은 관리자만 접근 가능하게 하세요.

## 택시팟 비즈니스 규칙
- 정원은 2~4명
- 생성자는 자동 참여
- 시작 시간이 지난 팟은 참여 불가
- 꽉 찬 팟은 참여 불가
- 사용자는 다른 활성 팟에 이미 참여 중이면 새 팟에 참여 불가
- 참여/취소/완료에 따라 상태를 일관되게 관리하세요

## 피드백 UX
원기획에는 “모임 시간 이후 1시간이 지나면 피드백 페이지가 강제로 뜬다”가 있지만, 웹 앱 특성상 사용자가 앱을 안 보고 있을 때 실제 강제 전환은 어렵습니다.
따라서 아래처럼 구현하세요.
- 피드백 기한이 도래한 팟을 계산한다
- 사용자가 앱 재방문 또는 관련 화면 진입 시
- 강한 배너 또는 모달로 피드백을 유도한다
- `/feedback/[partyId]` 페이지에서 후기/신고를 작성하게 한다

## 지도 UX
- 택시팟 생성 폼에 지도 검색 UI를 구현하세요.
- 출발지 선택 시 `place_name`, `lat`, `lng`, `departure_detail`을 저장할 수 있게 하세요.
- 도착지는 기본값을 `가톨릭대학교 성심교정`으로 두고, 필요 시 수정 가능하게 하세요.
- 택시팟 목록 필터는 최소한 아래 두 가지를 지원하세요.
  - 출발지 텍스트
  - 시간대 또는 날짜/시간
